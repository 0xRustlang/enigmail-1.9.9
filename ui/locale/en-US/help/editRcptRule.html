Using the Enigmail Rules Editor: Edit PGP Rule

In the Rules Editor, you can specify defaults per recipient for enabling encryption, signing and PGP/MIME, and to define what PGP key(s) to use. In this dialog, you can specify the rules for a single recipient, and for a group of recipients with very similar attributes.

  • Set PGP Rules for: Contains the email addresses of the recipients (without names, i.e. just an address like somebody@email.com). You can specify several email addresses, separated by spaces. The address specified here can consist of only the domain section so that mail to any address at that domain will be matched, e.g. @email.com will allow matching to body@email.com, somebody@email.com, anybody@email.com, etc.
  • Match begin: This modifies the matching of the address as seen from right. With this disabled, entering an address of body@email.com would also match somebody@email.com, anybody@email.com, etc. Enabling this will only match body@email.com. If multiple addresses are entered, the setting will apply to all.
  • Match end: This modifies the matching of the address as seen from left. With this disabled, entering an address of body@email.domain will match body@email.domain.com, body@email.domain.net, etc. Enabling this option will exactly match from the right side, e.g. @email.domain.com will only match @email.domain.com. If multiple addresses are entered, the setting will apply to all.

    With both Match begin and Match end enabled, the rule will only trigger on an exact email address match.

  • Key(s) to use: contains the PGP key ID's to use for the recipient(s). These can be selected from a list by clicking on the Select Key(s) button.
  • Default for Signing: enable or disable message signing. This either uses or overrides what you have specified in the message composition window. The values are:
    • Never: disable signing, even if it was enabled in the message composition window (overrules the other values)
    • Yes, if selected from in Message Composition: leave signing as specified in the message composition window
    • Always: enable signing, even if it was not enabled in the message composition window
These signing settings are applied for all rules that match. If one of the rules disables signing, the message will not be signed, regardless of other rules that specify Always.
  • Encryption: enable or disable message encryption. The allowed settings and their meaning are the same as for message signing.
  • PGP/MIME: enable or disable the use of the PGP/MIME (RFC 3156) message encoding. If PGP/MIME is disabled, the messages are encoded using "inline PGP". The allowed values and their meaning are the same as for message signing.

The rules are processed in the order displayed in the list in the PGP Rules Editor. Whenever a rule matches a recipient and contains a PGP Key ID, in addition to using the specified Key ID, the recipient is not considered anymore when processing further rules.

Note: The rule editor is not yet complete. It is possible to write some more advanced rules by directly editing the rules file (these rules should then not be edited anymore in the rule editor). Further information for directly editing the file is available on the Enigmail Homepage
