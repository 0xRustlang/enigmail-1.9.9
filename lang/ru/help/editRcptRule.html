Справка по Enigmail

Пользование редактором правил Enigmail

В редакторе правил можно указывать исходные значения настроек шифрования, подписи и PGP/MIME для каждого адресата, а также определять ключи, которые будут использоваться. В этом окне можно задавать правила для отдельных адресатов или групп адресатов со сходными атрибутами.

Установить правила Enigmail для адресата
Содержит адреса электронной почты получателей (без имен, напр., test@test.net). Можно указывать несколько адресов через пробел. Указанные здесь адреса могут состоять только из домена, так что подходить под правило будут любые адреса из этого домена, напр., @test.net будет указывать на test@test, proba@test.net и т.д.
Применить правило, если адресат ...
Изменяет подбор адресов. Если вводится несколько адресов, настройка будет применяться к всем. Нижеприведенные примеры предполагают, что в поле Установить правила Enigmail для адресата введено test@test.

Продолжить со следующего правила для подошедшего адреса
Это позволяет определять правило без указания идентификатора ключа в поле Использовать следующие ключи:, так что адрес электронной почты применяется для подбора ключа во время отправки. Кроме того, для тех же адресов будут обрабатываться дальнейшие правила.
Не проверять дальнейшие правила для подошедшего адреса
Для всех подошедших адресов обработка прекращается, т.е. процесс обработки переходит к следующему адресату.

Настройки по умолчанию для подписи
Подписывать или не подписывать. Это поле наследует или переопределяет то, что указано в окне создания сообщения. Возможные значения:
Никогда
Не подписывать, даже если задано в окне создания сообщения (отменяет другие значения)
Да, если выбрано при составлении сообщения
Делать, как указано в окне создания сообщения.
Всегда
Подписывать, даже если это не указано в окне создания сообщения

Эти настройки применяются для всех подошедших правил. Если одно из правил отменяет подпись, сообщение не будет подписываться, даже если в других правилах будет указано Всегда.

Шифрование
Шифровать или не шифровать сообщение. Набор значений такой же, как для подписи.
PGP/MIME
Включить или выключить кодирование сообщений в PGP/MIME (RFC 3156). Если выключено, сообщения кодируются как «PGP в тексте». Набор значений такой же, как для подписи.

Правила обрабатываются в порядке, приведенном в списке. Когда правило соответствует адресату и содержит идентификатор ключа, то кроме использования данного ключа адресат больше не участвует в дальнейшей обработке правил.

Подробнее о правилах можно прочесть на сайте Enigmail.
